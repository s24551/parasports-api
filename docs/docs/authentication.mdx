# Authentication & Security

## CORS Configuration
The API uses CORS (Cross-Origin Resource Sharing) to secure cross-origin requests.

```typescript
app.use(cors({
  origin: ['http://localhost:3000', 'http://localhost:4000'],
  methods: ['GET', 'POST', 'PUT', 'DELETE', 'PATCH'],
  allowedHeaders: ['Content-Type', 'Authorization']
}));
```
## Error Handling
All endpoints return appropriate HTTP status codes:
```
200: Success
201: Created
400: Bad Request
404: Not Found
500: Internal Server Error
```
## Headers
Each response includes the following headers:
- dla GET: X-Total-Count, X-Response-Time, Cache-Control
- dla POST: Location, X-Created-At, Cache-Control
- dla DELETE: X-Created-At, X-Deleted-At, X-Resource-IDCache-Control
